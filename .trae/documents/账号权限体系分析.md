# 账号权限体系分析

## 一、5 种权限及各自作用

| 权限代码 | 中文名 | 后端检查点 | Android 端用途 | 是否必须 |
|:---------|:-------|:-----------|:---------------|:--------:|
| `settings` | 设置管理 | `GET /api/users` `PUT/DELETE /api/users` `GET /api/kuaimai/credentials` `GET /api/kuaimai/session-status` `POST /api/kuaimai/update-credentials` `POST/DELETE /api/areas` | **同步快麦凭据到本地**（Worker 签名必备） | ✅ **PDA 必须** |
| `update_supplier` | 修改供应商 | `GET /api/kuaimai/suppliers` | ProductScreen 显示"切换供应商"按钮 | 可选 |
| `update_remark` | 修改备注 | **无后端检查** | ProductScreen 显示"修改备注"区域 | 可选 |
| `manage_area_image` | 库区图管理 | `POST /api/upload`(area) `DELETE /api/images`(area) | ProductScreen 显示"库区图管理"按钮 | 可选 |
| `manage_box_image` | 箱规图管理 | `POST /api/upload`(box) `DELETE /api/images`(box) | ProductScreen 显示"箱规图管理"按钮 | 可选 |

### 关键发现

**Worker 的 update_remark / update_supplier 操作是 Android 直连快麦 API**，不走后端。后端只检查 Worker 日志里报的"认证参数不完整"是本地凭据为空导致，根源是缺少 `settings` 权限导致 `syncKuaimaiCredentials()` 返回 403 静默失败。

**`update_remark` 在后端没有任何 endpoint 检查它**，这个权限纯粹控制 PDA 界面按钮显隐。

## 二、权限独立性验证

每个权限在后端 `auth.py` 中通过 `check_permission("xxx")` 独立检查，无层级依赖：

```python
VALID_PERMISSIONS = {"settings", "update_supplier", "update_remark", "manage_area_image", "manage_box_image"}

def check_permission(perm: str):
    def _check(user):
        if perm not in user["permissions"]:
            raise HTTPException(403)
        return user
    return _check
```

数据库 `user_permissions` 表中也是独立行存储，每行一个 `(user_id, permission)` 对。

## 三、各场景所需权限组合

| 场景 | 需要的权限 | 说明 |
|:-----|:-----------|:------|
| 登录后扫码取货 | **无需权限** | `orders.py` 全部用 `get_current_user`（仅需登录） |
| 改备注 | `settings` + `update_remark` | settings 用于同步凭据，update_remark 控制按钮显隐 |
| 切换供应商 | `settings` + `update_supplier` | settings 用于同步凭据，update_supplier 控制按钮显隐+供应商列表 |
| 上传库区图 | `settings` + `manage_area_image` | 同上 |
| 管理后台操作 | `settings` | 用户管理、拣货区管理、凭单管理 |

**结论：5 个权限完全独立，可按需分配。`settings` 是唯一 PDA 必须的权限（否则 Worker 无凭据签名）。其余 4 个权限各自控制对应功能，互不影响。**

## 四、推荐权限分配方案

| 账号 | 推荐权限 | 说明 |
|:----|:---------|:------|
| 管理员 | 全部 5 个 | 完全管理能力 |
| 取货员 | `settings` + `update_supplier` + `update_remark` | 能扫码、改备注、切供应商 |
| 拍照员 | `settings` + `manage_area_image` + `manage_box_image` | 只能扫码+拍照上传 |

## 五、不修改的文件

无需任何代码修改。权限系统设计正确，每个权限独立管理。用户只需在管理后台按需勾选。当前新账号的 `settings` 权限已在上一轮结论中确认为缺失，补上即可恢复 Worker 功能。
