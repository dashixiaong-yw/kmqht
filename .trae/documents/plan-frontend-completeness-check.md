## 一、功能缺失清单（按严重程度排序）

### P0 功能未实现（3个）

#### GAP-01: 设置页面完全未实现
- **原型**: 设置弹窗包含拣货区管理（添加/删除拣货区）、服务器地址配置、扫码方式配置
- **实际**: SettingsScreen.kt 仅显示"设置 - 待实现"占位文字，SettingsViewModel.kt 也是空壳
- **设计文档要求**: F23首次引导 + F28 API Key加密存储 + F8扫码反馈开关 + F6扫码方式选择
- **修复**: 实现完整的设置页面，包含：
  1. 拣货区管理（列表+添加+删除）
  2. 服务器地址配置
  3. API Key配置
  4. 扫码方式选择（PDA硬件/相机/手动）
  5. 扫码反馈开关（声音/振动）
  6. 版本信息

#### GAP-02: 首页缺少首次使用引导提示（F23）
- **原型**: 首页有黄色提示条"首次使用？点击设置配置服务器地址和扫码方式"，可关闭
- **实际**: HomeScreen.kt 无此引导提示
- **修复**: 在HomeScreen中添加首次使用引导提示条，使用DataStore记录是否已显示过

#### GAP-03: 取货单详情页待办行缺少规格图和库区图/装箱图
- **原型**: 每个待办行左侧有52dp规格图（底部标注"规格图"），右侧有40dp库区图+装箱图小方块
- **实际**: PickItemRow.kt 只有文字信息（规格名+供应商名+完成按钮），完全缺少图片显示
- **设计文档**: "待办行显示内容：规格图（方块图片+底部'规格图'文字标注）、库区图、箱图"
- **修复**: PickItemRow添加：
  1. 左侧52dp规格图（带底部"规格图"标注，点击跳转商品详情）
  2. 右侧40dp库区图+装箱图小方块（带底部文字标注，点击弹窗查看大图）

### P1 功能部分实现（5个）

#### GAP-04: 取货单详情页缺少下拉刷新（F15）
- **原型/文档**: 支持下拉刷新同步多PDA操作
- **实际**: PickDetailScreen.kt 无下拉刷新组件，ViewModel有refresh()方法但Screen未调用
- **修复**: 用PullToRefreshBox包裹LazyColumn

#### GAP-05: 取货单详情页缺少屏幕常亮（F13）
- **文档**: 取货单详情页和商品详情页需保持屏幕常亮
- **实际**: ProductScreen.kt已实现KeepScreenOn，但PickDetailScreen.kt未实现
- **修复**: 在PickDetailScreen中添加KeepScreenOn

#### GAP-06: 取货单详情页缺少待办行删除功能（F20/F25）
- **原型**: 右键(长按)待办行弹出删除确认
- **实际**: PickDetailScreen的长按回调为`onLongPress = { /* TODO: 操作菜单 */ }`，未实现
- **修复**: 长按弹出操作菜单（删除、查看商品详情），删除需二次确认

#### GAP-07: 取货单详情页缺少待办行自动排序
- **设计文档**: "未完成的按添加时间倒序（最新在上），已完成的按添加时间倒序排在未完成组下方"
- **实际**: PickDetailScreen只按供应商过滤，未按状态+时间排序
- **修复**: 在filteredItems后添加排序逻辑：先按status(0在前1在后)，同status按createdAt倒序

#### GAP-08: 取货单详情页缺少扫码成功/失败反馈（F8）
- **设计文档**: 扫码成功→短振动+提示音；失败→长振动+错误音；重复→中等振动
- **实际**: 只有重复扫码振动，缺少成功/失败的振动+声音反馈
- **修复**: 在onBarcodeScanned成功后添加短振动+提示音，失败时添加长振动+错误音

### P2 UI与原型不一致（7个）

#### GAP-09: 首页布局与原型不一致
- **原型**: 居中Logo+副标题"扫码取货·高效管理"，3个模块卡片水平排列图标+标题+描述
- **实际**: TopAppBar蓝色背景+白色标题，3个卡片居中排列（图标在上方），缺少Logo和副标题
- **修复**: 
  1. 去掉TopAppBar，改为居中Logo区域
  2. 添加副标题"扫码取货·高效管理"
  3. 模块卡片改为水平布局（图标在左，标题+描述在右），匹配原型

#### GAP-10: 取货单详情页供应商筛选Chips显示条件不对
- **原型**: 供应商筛选始终显示（全部+各供应商）
- **实际**: `if (suppliers.size > 2)` 才显示，当只有1个供应商时不显示
- **修复**: 去掉`suppliers.size > 2`条件，始终显示供应商筛选

#### GAP-11: 取货单详情页底部进度显示与原型不一致
- **原型**: 底部显示"进度 3/5"文字 + "全部完成"按钮
- **实际**: 有进度条+数字+按钮，但文字格式为"$completedCount/$totalCount"而非"进度 X/Y"
- **修复**: 进度文字改为"进度 $completedCount/$totalCount"

#### GAP-12: 取货单详情页扫码框placeholder不一致
- **原型**: "按PDA扫码键扫描规格编码"
- **实际**: "扫码或输入条码"
- **修复**: 改为"按PDA扫码键扫描规格编码"

#### GAP-13: 商品详情页备注保存按钮颜色不一致
- **原型**: 保存按钮为浅蓝底+深蓝字（主操作色）
- **实际**: 保存按钮为浅绿底+深绿字（完成色）
- **修复**: 保存按钮改为PrimaryLightBg+PrimaryLightText

#### GAP-14: 商品详情页图片上传缺少提示文字
- **原型**: 图片上传下方有"点击上传/替换 · 右键删除 · 上传前自动压缩"提示
- **实际**: 无此提示文字
- **修复**: 在ImageUploadGrid下方添加提示文字

#### GAP-15: 取货单卡片缺少进度点（dots）
- **原型**: 每个取货单卡片底部有进度点（绿色=已完成，灰色=未完成）
- **实际**: PickOrderCard.kt只有文字进度"进度: 3/5"，无进度点
- **修复**: 在PickOrderCard底部添加进度点指示器
