# 新创建 PDA 账号 Worker 签名错误排查方案

## 一、问题描述

更换快麦凭证后，**旧账号**可正常操作（改备注、切供应商），**新创建的账号** Worker 日志报 `msg=签名错误 认证参数不完整`。

## 二、当前状态分析

### 2.1 Worker 调用链路

```
新账号登录
  ↓
syncKuaimaiCredentials()
  → GET /api/kuaimai/credentials
  → 需要 settings 权限（auth.py L219）
  ↓
结果A: 403 无权限 → catch 吞掉 → 本地凭据为空
结果B: 200 有凭据 → 存入 EncryptedSharedPreferences
  ↓
Worker 执行 update_remark/update_supplier
  → KuaimaiApiService → KuaimaiInterceptor（读取本地凭据）
  ↓
凭据为空 → 跳过签名（KuaimaiInterceptor.kt L45-L48）
  → 请求无 appKey/timestamp/session/sign 参数
  → 快麦 API 返回 "认证参数不完整"
```

**关键证据**：
- [KuaimaiInterceptor.kt L45-48](file:///d:/trea项目/快麦取货通/app/src/main/java/com/kuaimai/pda/data/api/KuaimaiInterceptor.kt#L45-L48)：凭据为空时跳过签名，但不阻断请求 → 快麦 API 收到无认证参数的请求 → "认证参数不完整"
- [UserRepository.kt L318-335](file:///d:/trea项目/快麦取货通/app/src/main/java/com/kuaimai/pda/data/repository/UserRepository.kt#L318-L335)：`syncKuaimaiCredentials()` 在 catch 中静默吞掉异常（包括 403），凭据为空则跳过写入
- [auth.py L219-228](file:///d:/trea项目/快麦取货通/backend/app/auth.py#L219-L228)：`GET /api/kuaimai/credentials` 需要 `settings` 权限

### 2.2 旧账号为什么能工作

旧账号在**凭证更换之前**已经执行过 `syncKuaimaiCredentials()`，本地 `EncryptedSharedPreferences` 中存有**旧凭证**。旧凭证的 session 未过期（30天有效），Worker 用旧凭证签名请求 → 快麦 API 正常响应。

### 2.3 新账号为什么失败

新账号是凭证更换后才创建的，登录时执行 `syncKuaimaiCredentials()`：

- **场景A（最可能）**：新账号没有 `settings` 权限 → 403 → 静默失败 → 本地凭据为空 → Worker 无签名 → "认证参数不完整"
- **场景B（也可能）**：新账号有 `settings` 权限，但 `backend/kuaimai.json` 中 session 与新 appKey 不匹配 → 签名错误

## 三、修复方案

### 3.1 管理后台检查新账号权限（第一步）

访问 `http://服务器IP:8900/admin` → 用户管理 → 检查新创建账号：
- 是否勾选了 `settings` 权限？
- 如果没有，勾选并保存

### 3.2 验证新账号能否操作（第二步）

1. PDA 退出当前新账号登录
2. 重新登录
3. 尝试修改备注或切换供应商
4. 如果成功 → 问题解决（根因：缺少 settings 权限）

### 3.3 如果仍有签名错误（第三步）

如果确认有 `settings` 权限但仍报签名错误，说明新凭证本身无效（session 与 appKey 不匹配）。

解决方案：
1. 通过管理后台 → 凭证管理，重新获取并填入有效的 session/refreshToken
2. 或联系快麦开放平台确认凭证有效性

### 3.4 旧账号处理（第四步）

旧账号虽然能操作，但使用的是旧凭证。旧 session 30天到期后也会失效。

到期前操作流程：
1. PDA 旧账号退出登录
2. 重新登录 → 触发 `syncKuaimaiCredentials()` → 本地凭证更新为新凭证
3. 如果新凭证有效，继续正常使用

## 四、不修改的文件

| 文件 | 原因 |
|:----|:-----|
| `KuaimaiInterceptor.kt` | 跳过签名逻辑是正确行为（空凭据不签名） |
| `UserRepository.kt` | catch 静默是设计意图（不阻塞登录流程） |
| `backend/kuaimai.json` | 文件路径修复已在 v1.88 完成 |
| `app/build.gradle.kts` | 无需修改代码 |
