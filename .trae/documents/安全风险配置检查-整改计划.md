# 安全风险配置检查与整改计划

## 目标

消除「快麦取货通」APK 安装时系统提示的"未知来源应用风险"警告，减少定制行业 PDA 系统对 debug 证书签名的安全标记。

## 成功标准

1. APK 使用自定义 release 证书签名（非 Android SDK 默认 debug 证书）
2. 代码混淆已启用（R8 压缩混淆）
3. 构建后的 release APK 安装时不再出现"风险"类额外提示（仅保留通用的"未知来源"提示）
4. 构建验证通过（`./gradlew lint` + `assembleDebug` 均通过）

## 当前状态分析

| 问题 | 现状 | 严重程度 |
|------|------|----------|
| 签名证书 | 使用 Android SDK 默认 debug 证书（公开通用，易被标记为高风险） | **核心问题** |
| 代码混淆 | `isMinifyEnabled = false`，APK 未混淆，易反编译 | 次要 |
| 明文 HTTP 请求 | 无 `networkSecurityConfig`，Android 9+ 明文流量可能被拦截 | 需配套修复 |
| proguard 规则文件 | `proguard-rules.pro` 不存在，即使开启混淆也会报错 | 需修复 |

## 修改方案

### 1. 创建 release 签名证书

执行 `keytool` 生成自签名 keystore：

- 文件路径: `app/kuaimai-release.keystore`
- 有效期: 25 年（内部使用足够）
- 别名: `kuaimai`
- 密码: 统一密码（记录在构建脚本中）

### 2. 配置 app/build.gradle.kts

在 `android` 块内添加 `signingConfigs` 配置：

```kotlin
android {
    signingConfigs {
        create("release") {
            storeFile = file("kuaimai-release.keystore")
            storePassword = "kuaimai2024"
            keyAlias = "kuaimai"
            keyPassword = "kuaimai2024"
        }
    }
    // ...
    buildTypes {
        release {
            isMinifyEnabled = true
            proguardFiles(
                getDefaultProguardFile("proguard-android-optimize.txt"),
                "proguard-rules.pro"
            )
            signingConfig = signingConfigs.getByName("release")
        }
        debug {
            // debug 沿用默认 debug 证书，不做改动
        }
    }
}
```

### 3. 创建 proguard-rules.pro

在 `app/` 目录下创建，保留 Hilt、Retrofit、Room 等框架的必要规则：

```
# Hilt
-keep class dagger.hilt.** { *; }
-keep class javax.inject.** { *; }
-keep class * extends dagger.hilt.android.internal.managers.ViewComponentManager$FragmentContextWrapper { *; }

# Retrofit
-keepattributes Signature, InnerClasses, EnclosingMethod
-keepattributes RuntimeVisibleAnnotations, RuntimeVisibleParameterAnnotations
-keepclassmembers,allowshrinking,allowobfuscation interface * {
    @retrofit2.http.* <methods>;
}
-dontwarn org.codehaus.mojo.animal_sniffer.IgnoreJRERequirement
-dontwarn javax.annotation.**
-dontwarn kotlin.Unit
-dontwarn retrofit2.KotlinExtensions
-dontwarn retrofit2.KotlinExtensions$*

# Gson
-keepattributes Signature
-keepattributes *Annotation*
-keep class com.kuaimai.pda.data.model.** { *; }
-keep class com.kuaimai.pda.data.dto.** { *; }

# Room
-keep class * extends androidx.room.RoomDatabase
-dontwarn androidx.room.paging.**

# OkHttp
-dontwarn okhttp3.**
-dontwarn okio.**

# Coil
-dontwarn coil.**

# CameraX
-keep class androidx.camera.** { *; }
```

### 4. 创建网络安全配置文件

新增 `app/src/main/res/xml/network_security_config.xml`：

```xml
<?xml version="1.0" encoding="utf-8"?>
<network-security-config>
    <base-config cleartextTrafficPermitted="true">
        <trust-anchors>
            <certificates src="system" />
        </trust-anchors>
    </base-config>
</network-security-config>
```

在 AndroidManifest.xml 中引用：
```xml
<application
    ...
    android:networkSecurityConfig="@xml/network_security_config"
    ...
>
```

### 5. 验证

1. `./gradlew lint` 通过
2. `./gradlew assembleRelease` 构建成功
3. 确认 APK 输出为 `快麦取货通-{version}.apk`

## 涉及文件清单

| 文件 | 操作 | 说明 |
|------|------|------|
| `app/kuaimai-release.keystore` | **新建** | keytool 生成 |
| `app/proguard-rules.pro` | **新建** | R8 混淆规则 |
| `app/src/main/res/xml/network_security_config.xml` | **新建** | 网络安全配置 |
| `app/build.gradle.kts` | **修改** | 添加 signingConfigs、启用混淆 |
| `app/src/main/AndroidManifest.xml` | **修改** | 引用 networkSecurityConfig |

## 风险与注意事项

1. **keystore 请妥善保存**：丢失后无法用同一签名升级安装旧版 APK
2. **该方案是内部测试证书**：非 Google Play 认可的正式签名，仅适用于企业内部分发
3. **混淆可能引发运行时崩溃**：如有异常，需补充 proguard 规则
4. **network_security_config.xml 允许了明文 HTTP**：这是内网 PDA 场景的合理配置，生产环境应限制到特定域名
