## Bug 2：快麦图片不显示 — 根因确认

用户提供快麦图片URL格式：`https://img.alicdn.com/bao/uploaded/...`

**结论**：`skuPicPath` 是完整URL，后端保存正确。

**根因**：PDA 通过 SakuraFRP 连接后端，可能**无法直接访问** `img.alicdn.com`（阿里CDN）。ProductScreen 的 area/box 图片使用后端 `/images/` 静态文件服务，而 PickItemRow 的 `sku picPath` 直接指向外网 CDN。

**修复**：后端新增 `/api/images/proxy` 代理路由，PDA 通过后端加载快麦图片。

```
修复前: PDA → AsyncImage(picPath="https://img.alicdn.com/...") → 无法访问
修复后: PDA → AsyncImage(picPath="http://后端地址/api/images/proxy?url=...") → 后端下载 → PDA加载
```

### 2a. 后端新增图片代理路由

[images.py](file:///d:/trea项目\快麦取货通/backend/app/routers/images.py) 新增：

```python
import httpx

@router.get("/proxy")
def proxy_image(url: str = Query(..., description="快麦图片URL")):
    """代理加载快麦图片（PDA无法直连阿里CDN时使用）"""
    try:
        response = httpx.get(url, timeout=10.0)
        response.raise_for_status()
        return Response(content=response.content, media_type=response.headers.get("content-type", "image/jpeg"))
    except Exception as e:
        logger.error(f"图片代理失败: {url} - {e}")
        raise HTTPException(status_code=502, detail="图片加载失败")
```
