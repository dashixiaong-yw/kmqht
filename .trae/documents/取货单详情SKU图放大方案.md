# 取货单详情商品 SKU 图放大方案

## 一、问题分析

取货单详情每行左侧的 SKU 商品缩略图当前尺寸为 **52dp × 52dp**（`PickItemRow.kt` L87）。用户反馈 PDA 屏幕尚有空间，希望放大一倍左右。

### 当前布局

```
┌──────────────────────────────────────────────────────┐
│ ┌──────┐          │                  │ ┌────┐ ┌────┐ │
│ │ 52x52 │  propertiesName            │ │ 44 │ │ 44 │ │
│ │  SKU  │  supplierName              │ │库区 │ │箱图│ │
│ └──────┘          │                  │ └────┘ └────┘ │
└──────────────────────────────────────────────────────┘
```

---

## 二、修改内容

### 文件：`PickItemRow.kt` L87 — 唯一改动

**改动**：`size(width = 52.dp, height = 52.dp)` → `size(width = 100.dp, height = 100.dp)`

```kotlin
// 修改前
.size(width = 52.dp, height = 52.dp)

// 修改后
.size(width = 100.dp, height = 100.dp)
```

### 效果预览（PDA 典型分辨率 540×960 / 720×1280）

```
修改前:                         修改后:
┌──────┐                       ┌──────────┐
│52x52 │  propertiesName ────  │  100x100 │  propertiesName ────
│      │  supplierName         │   SKU    │  supplierName
└──────┘                       │          │
                                └──────────┘
```

- 右侧文本区（`Column.weight(1f)`）自动收缩，保持布局响应
- 最右侧 44dp 库区图/箱图保持不变

### 为什么不改其他尺寸

| 元素 | 当前尺寸 | 处理 |
|:-----|:--------|:-----|
| SKU 商品图（picPath） | 52×52dp | ✅ 100×100dp（用户需求） |
| 库区图/箱图缩略图 | 44×44dp | ❌ 用户未要求变化 |
| `propertiesName` 字体 | 16sp | 不变 |
| `supplierName` 字体 | 20sp | 不变 |

---

## 三、回归风险

| 风险 | 分析 | 等级 |
|:-----|:-----|:-----|
| 5 英寸 PDA 屏幕空间 | 100dp 约等于 2cm，宽高各增加约 1cm，仍远小于 PDA 宽度 | 低 |
| 文本换行 | 文本区使用 `weight(1f)`，自动适应 | 无影响 |
| 库区图/箱图遮挡 | 保持不变，不受影响 | 无影响 |
| 每行高度 | 从 52dp 增加到 100dp，列表滚动长度增加约一倍，但 PDA 屏幕充足 | 可接受 |

---

## 四、验证步骤

1. `./gradlew lint` — 无新增警告
2. `./gradlew assembleRelease` — 构建 APK
3. 安装到 PDA → 进入取货单详情页
4. 确认 SKU 图尺寸约为原来的 2 倍
5. 确认图片清晰度（Coil 自动缩放 100dp 内的 crop）
6. 确认文本内容正常显示、库区图箱图不变
