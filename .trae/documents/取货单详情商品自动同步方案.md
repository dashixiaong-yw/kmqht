# 取货单详情商品列表自动同步方案

## 一、问题描述

同一账号切换 PDA 登录后，之前创建的取货单详情中的商品无法显示。同样，发布到公共的订单被其他 PDA 领取后也无法显示商品。因为取货单详情页的商品列表仅从本地 Room 数据库读取，`init` 块中未触发后端同步。

## 二、当前状态分析

### 2.1 数据流（当前）

```
PickDetailScreen 打开
  └─ PickDetailViewModel.init
      ├─ loadOrder()           → Room 本地读取（有数据）
      ├─ loadSuppliers()       → 后端 API 获取供应商列表
      └─ ❌ 缺失：syncItemsFromBackend()
            → 商品列表一直为 emptyList()
```

### 2.2 现有基础设施

| 项目 | 状态 | 说明 |
|:-----|:-----|:------|
| `syncItemsFromBackend()` | ✅ 已存在 | `PickDetailViewModel.kt` L396，已实现完整的后端→本地 upsert 逻辑 |
| `getOrderDetail` API | ✅ 已存在 | `orderApiService.getOrderDetail(token, orderId)` 返回完整明细列表 |
| items 去重逻辑 | ✅ 已存在 | 按 `skuOuterId` 匹配，已有则更新字段，无则插入 |
| 异常处理 | ✅ 已存在 | `syncItemsFromBackend()` 内部有 try-catch，失败仅 log 不崩溃 |

### 2.3 根因

`PickDetailViewModel.init` (L100-103) 仅调用了 `loadOrder()` 和 `loadSuppliers()`，未调用 `syncItemsFromBackend()`：

```kotlin
init {
    loadOrder()
    loadSuppliers()
}
```

当切换 PDA 后，本地 Room 无该订单的明细数据，而 `items` StateFlow（L60-62）仅监听 Room 数据库变化，因此一直为空列表。只有用户手动下拉刷新（调用 `refresh()`）后才能显示。

## 三、修改方案（2个文件）

### 3.1 修复：PickDetailViewModel — init 时自动同步取货明细

**文件**：`app/src/main/java/com/kuaimai/pda/ui/pickdetail/PickDetailViewModel.kt`

### 3.2 具体变更

**修改位置**：L100-103 `init` 块

**修改前：**

```kotlin
init {
    loadOrder()
    loadSuppliers()
}
```

**修改后：**

```kotlin
init {
    loadOrder()
    loadSuppliers()
    viewModelScope.launch {
        syncItemsFromBackend()
    }
}
```

**说明**：
- `syncItemsFromBackend()` 是 `suspend` 函数，需要用 `viewModelScope.launch` 包装
- `loadOrder()` 和 `loadSuppliers()` 已经在内部使用 `viewModelScope.launch`，不需要改动
- `syncItemsFromBackend()` 不依赖 `loadOrder()` 或 `loadSuppliers()` 的结果，可独立执行

### 3.2 新增：PickListViewModel + PickListScreen — 取货单列表页下拉刷新

**文件A**：`app/src/main/java/com/kuaimai/pda/ui/picklist/PickListViewModel.kt`

**修改**：`loadActiveOrders()` 增加 loading 状态管理，供 PullToRefreshBox 驱动刷新指示器。

```kotlin
private fun loadActiveOrders() {
    viewModelScope.launch {
        _isLoading.value = true
        try {
            val token = userRepository.getToken()
            val response = orderApiService.listOrders(token, status = 0)
            _activeOrders.value = response.data.map { it.toOrderEntity() }
        } catch (e: Exception) {
            Log.e("PickListViewModel", "加载取货单列表失败: ${e.message}", e)
            _errorMessage.value = "加载取货单列表失败: ${e.message?.take(80) ?: "未知错误"}"
            _activeOrders.value = emptyList()
        } finally {
            _isLoading.value = false
        }
    }
}
```

**文件B**：`app/src/main/java/com/kuaimai/pda/ui/picklist/PickListScreen.kt`

**改变A** — 新增 import（L1-50 区域）：

```kotlin
import androidx.compose.material3.pulltorefresh.PullToRefreshBox
```

**改变B** — 用 `PullToRefreshBox` 包裹列表内容（L161-224 区域）：

```kotlin
// 修改前：
} else {
    // 进行中列表
    Column(
        modifier = Modifier
            .fillMaxSize()
            .padding(innerPadding)
    ) {
        if (activeOrders.isEmpty()) {
            ...
        } else {
            LazyColumn(...) { items(...) { PickOrderCard(...) } }
        }

}

// 修改后：
} else {
    // 进行中列表
    PullToRefreshBox(
        isRefreshing = isLoading,
        onRefresh = { viewModel.refreshActiveOrders() },
        modifier = Modifier
            .fillMaxSize()
            .padding(innerPadding)
    ) {
        Column(modifier = Modifier.fillMaxSize()) {
            if (activeOrders.isEmpty()) {
                ...
            } else {
                LazyColumn(...) { items(...) { PickOrderCard(...) } }
            }
            TextButton("查看已完成...")
        }
    }
}
```

> 注意：`innerPadding` 从 `Column` 上移到 `PullToRefreshBox`，`Column` 不再独立设置 padding。

## 四、流程示意（修复后）

```
PickDetailScreen 打开
  └─ PickDetailViewModel.init
      ├─ loadOrder()           → Room 本地读取（订单标题/状态正常显示）
      ├─ loadSuppliers()       → 后端 API 获取供应商列表
      └─ syncItemsFromBackend()
           ├─ orderApiService.getOrderDetail(token, orderId)
           ├─ 遍历 detail.items
           │   ├─ 本地不存在 → upsert 插入本地 Room
           │   └─ 本地已存在 → 更新不可变字段 + 状态同步
           └─ Room Flow 自动更新 → UI 显示商品列表
```

## 五、回归风险评估

| 风险点 | 评估 |
|:-------|:-----|
| 网络异常 | `syncItemsFromBackend()` 内部已有 try-catch，失败仅 log 不崩溃，用户仍可下拉重试 |
| 并发竞态（用户同时扫码同步） | `pick_item` 表有 `UNIQUE(order_id, sku_outer_id)` 约束（L18），`ON CONFLICT REPLACE` 自动处理，不会产生重复记录 |
| 初始化性能 | HTTP 调用在 `viewModelScope` 的协程中执行，不阻塞 UI 线程 |
| 接口返回空列表 | 本地不变（仍为空），UI 显示"暂无取货明细"，行为一致 |
| 离线模式 | API 失败后静默处理（仅 log），不影响已有本地数据 |
| 供应商列表同步 | `syncItemsFromBackend()` 尾部已调用 `loadSuppliers()`（L428），同步商品后自动刷新供应商过滤 |
