## 二、根因

### 完整触发链（最终确认）

```
APP 更新安装后:
  → EncryptedSharedPreferences 读取旧token → 返回空字符串
    （Keystore 不稳定 / 加密数据不可恢复）
  → _cachedToken = ""
  → getToken() → ""

用户点击取货列表 / 扫码:
  → API 调用 Header: X-User-Token = ""
  → 后端 get_current_user → token为空 → HTTP 401

TokenAuthenticator:
  → userToken = prefs.getString(KEY_USER_TOKEN) = ""  （空）
  → 仍然调用 systemApiService.refreshSession("") 
    → POST /api/kuaimai/refresh-session  Header: X-User-Token = ""
    → 后端验证 token → 空值 → 401
    → Retrofit 抛 HttpException
  → refreshResult = null
  → notifySessionExpired()

AppNavigation:
  → SessionExpiredEvent.isExpired = true
  → 弹窗: "快麦API会话已过期，请在Web管理后台重新授权"
  → 用户点击"知道了" → **仅关闭弹窗，不做任何处理**
  → 用户仍停留在当前页面，所有功能不可用
  → 只能关闭APP重开或退出登录
```

### 核心问题

> **弹窗没有引导用户重新登录**。"知道了"按钮仅关闭弹窗，用户状态仍处于"半登录"状态（token 为空但未跳转登录页）。

重新登录后 `_cachedToken` + `syncKuaimaiCredentials` 都能正常恢复——唯一缺失的是从弹窗通往登录页的路径。

## 三、修改方案

### 仅 1 个文件：`AppNavigation.kt`

#### 修改 1：弹窗标题/文本/按钮 — 全部改为登录相关

```kotlin
// 修改前（L195-210）
title = { Text("快麦会话已过期") },
text = { Text("快麦API会话已过期，请在Web管理后台重新授权\n（请用电脑浏览器访问管理后台）") },
confirmButton = {
    TextButton(onClick = { showSessionExpiredDialog = false; SessionExpiredEvent.reset() }) {
        Text("知道了")
    }
},
dismissButton = {
    TextButton(onClick = {
        showSessionExpiredDialog = false
        SessionExpiredEvent.reset()
        navController.navigate(Routes.SETTINGS)
    }) {
        Text("前往设置")
    }
}

// 修改后
title = { Text("登录过期") },
text = { Text("当前登录已失效，请重新登录") },
confirmButton = {
    TextButton(onClick = {
        showSessionExpiredDialog = false
        SessionExpiredEvent.reset()
        navController.navigate(Routes.LOGIN) {
            popUpTo(navController.graph.startDestinationId) { inclusive = true }
        }
    }) {
        Text("重新登录")
    }
}
// 移除 dismissButton — 用户只能点击"重新登录"
```

#### 修改 2：`onDismissRequest` — 也跳转登录

```kotlin
// 修改前
onDismissRequest = { showSessionExpiredDialog = false; SessionExpiredEvent.reset() }

// 修改后
onDismissRequest = {
    showSessionExpiredDialog = false
    SessionExpiredEvent.reset()
    navController.navigate(Routes.LOGIN) {
        popUpTo(navController.graph.startDestinationId) { inclusive = true }
    }
}
```

## 五、数据流验证

```
token 为空 → API 401 → TokenAuthenticator → notifySessionExpired()
  → 弹窗显示
  → 用户点击"重新登录"
    → navController.navigate(Routes.LOGIN) { popUpTo(startDestinationId) { inclusive = true } }
    → 清除返回栈 → 仅显示登录页
  → 用户登录
    → login() → _cachedToken = newToken ✅
    → syncKuaimasiCredentials(newToken) → 获取快麦凭证 ✅
    → 所有 API 正常 ✅
```

## 六、回归风险

| # | 风险 | 说明 | 等级 |
|:-:|:-----|:------|:----:|
| 1 | 弹窗后跳转登录，用户需重新输入密码 | 一次性的，后续 token 有效不会再弹窗 | 低 |
| 2 | 极少数情况快麦 session 真过期时也会跳登录 | 但后台自动刷新确保不过期；且重新登录会同步新凭证 | 低 |
| 3 | `navigate(Routes.LOGIN)` 与已有 `loginRequired` 监听冲突 | 不会——`loginRequired` 由 `handleAuthError` 触发，此路径不走 handleAuthError | ✅ |
