#### 新旧格式混存说明

旧数据库中的订单（如 `20260619-B区1`）和新订单（`B区-20260619-1`）会共存，排序表现：

| 订单 | 旧排序方式 (`substringAfter`) | 新排序方式 (`substringBefore`) |
|------|:---:|:---:|
| 旧订单 `20260619-B区1` | 按 `B区1` 排序 | 按 `20260619` 排序（归为一组） |
| 新订单 `B区-20260619-1` | 按 `20260619-1` 排序（乱） | 按 `B区` 排序（正确） |

**结论**：新旧混存期间，旧订单会按日期值 `20260619` 归为一组显示在列表末尾，不影响功能。30天后旧订单自动清理完毕。

### 改动 2：创建后自动跳转详情页

#### 导航链路

```
用户点击拣货区按钮
    ↓
PickListScreen onConfirm → viewModel.createOrder(areaName)
    ↓
ViewModel: API调用 → 成功 → emit navigateToOrderEvent(response.id)
    ↓
PickListScreen: LaunchedEffect 收集事件 → onNavigateToDetail(orderId)
    ↓
创建成功，自动进入详情页
```

#### 风险场景分析

| 场景 | 分析 | 安全？ |
|------|------|:----:|
| API 调用失败 | `catch` 块中不 emit 事件，不回导航 | ✅ |
| 用户快速返回主页 | `LaunchedEffect` 随 Compose 生命周期自动取消 | ✅ |
| 事件被消费两次 | `SharedFlow` 无 replay，每个事件只被消费一次 | ✅ |
| 导航回调异常 | 和点击卡片一样的 `onNavigateToDetail` 回调，已有保障 | ✅ |
| 页面重建 | `LaunchedEffect(Unit)` 重新收集，但旧事件没有 replay | ✅ |
| 并发多次创建 | 每个 `createOrder` 独立发射事件，顺序消费 | ✅ |
