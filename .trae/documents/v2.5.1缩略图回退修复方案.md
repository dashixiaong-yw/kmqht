# v2.5.1 库区图/箱图缩略图不显示修复方案

## 一、根因分析

### 完整链路追查

```
后端上传图片:
  → 保存原始文件: IMAGE_DIR/20260624/AYGYC_area_96c9355c.jpg
  → _generate_thumbnail(原图路径) → 生成: IMAGE_DIR/20260624/AYGYC_area_96c9355c_thumb.jpg
    → 如果 Pillow 未安装 / 磁盘权限不足 / 图片格式问题 → 静默失败（try-catch，仅打 warning 日志）
  → DB 记录: file_path = "20260624/AYGYC_area_96c9355c.jpg"
  → API 返回: image_url = "images/20260624/AYGYC_area_96c9355c.jpg"

前端加载:
  → getImageUrls(skuOuterId)
    → areaUrl = "https://frp-off.com:64623/images/20260624/AYGYC_area_96c9355c.jpg"   ✅ 文件存在
    → areaThumbUrl = "https://frp-off.com:64623/images/20260624/AYGYC_area_96c9355c_thumb.jpg"

PickItemRow:
  → 检查条件: if (!areaImageUrl.isNullOrEmpty())  ← 检查的是完整图URL ✅
  → 实际加载: AsyncImage(model = areaThumbUrl)     ← 加载的是缩略图URL ❌
    → 服务器返回 404（缩略图文件不存在）
    → Coil 静默失败（无 fallback）
    → 44x44 格子显示灰色背景，用户看不到任何内容

点击后:
  → onAreaImageClick → previewImageUrl = areaImageUrl（完整图）
  → 大图预览弹窗 → 加载完整图 → 显示正常 ✅
```

### 根因总结

| # | 因素 | 说明 |
|:-:|:-----|:------|
| 1 | 缩略图文件不存在 | 上传时 `_generate_thumbnail` 可能因 Pillow 配置/权限静默失败 |
| 2 | 旧图片没有缩略图 | 早期版本没有缩略图逻辑，已上传的图片从未生成过 `_thumb.jpg` |
| 3 | 前端无 fallback | `AsyncImage` 加载失败时显示空白，没有回退到完整图 |

---

## 二、修复方案

### 方案：前端增加缩略图加载失败回退

**原理**：`AsyncImage` 加载 `_thumb.jpg` 失败时，自动回退加载完整图 URL。Coil 会缓存完整图，后续加载直接命中缓存，体验与缩略图无异。

### 修改 1：`PickItemRow.kt` — AsyncImage 增加 fallback

```kotlin
// 修改前（L146-152）
if (!areaImageUrl.isNullOrEmpty()) {
    AsyncImage(
        model = areaThumbUrl,
        contentDescription = "库区图",
        modifier = Modifier.fillMaxSize(),
        contentScale = ContentScale.Crop
    )
}

// 修改后
if (!areaImageUrl.isNullOrEmpty()) {
    AsyncImage(
        model = areaThumbUrl ?: areaImageUrl,
        contentDescription = "库区图",
        modifier = Modifier.fillMaxSize(),
        contentScale = ContentScale.Crop
    )
}
```

同样修改箱图（L164-170）：

```kotlin
// 修改前
if (!boxImageUrl.isNullOrEmpty()) {
    AsyncImage(
        model = boxThumbUrl,
        ...
    )
}

// 修改后
if (!boxImageUrl.isNullOrEmpty()) {
    AsyncImage(
        model = boxThumbUrl ?: boxImageUrl,
        ...
    )
}
```

**效果**：
- 有缩略图时 → 加载缩略图（200px，约10KB）
- 无缩略图时 → 加载完整图（Coil 自动缩放到 44dp 显示，后续缓存命中）
- 两边都没有时 → 不显示（保持了原有的 `SurfaceGray` 背景）

### 修改 2：`PickDetailViewModel.kt` — buildThumbUrl 增加后端生成（可选增强）

在 `getImageUrls()` 中，当 thumb URL 为 null 时直接用 full URL：

```kotlin
ImageUrls(
    areaUrl = areaUrl,
    boxUrl = boxUrl,
    areaThumbUrl = buildThumbUrl(areaUrl) ?: areaUrl,  // ← thumb为空时用full
    boxThumbUrl = buildThumbUrl(boxUrl) ?: boxUrl      // ← thumb为空时用full
)
```

但此修改是冗余的（因为 `buildThumbUrl` 只有在 `fullUrl == null` 时才返回 null），所以**不需要改 ViewModel**。PickItemRow 中的 `?:` 回退已经足够。

## 四、数据流验证

```
PickItemRow 渲染:
  → areaThumbUrl = "https://...xxx_thumb.jpg"
  → areaImageUrl = "https://...xxx.jpg"

[有缩略图]
  → AsyncImage(model = areaThumbUrl) → 加载成功 → 显示缩略图 ✅

[无缩略图]
  → AsyncImage(model = null ?: areaImageUrl) → 加载完整图
    → Coil trustAllClient 请求 → 200 OK
    → 自动缩放到 44dp 显示
    → 内存缓存后后续秒开 ✅

[两者都无]
  → AsyncImage(model = null) → 不显示 → SurfaceGray 背景 ✅
```

## 五、回归风险

| # | 风险 | 说明 | 等级 |
|:-:|:-----|:------|:----:|
| 1 | 完整图较大（最大2MB），首次加载慢 | Coil 缓存后秒开，仅首次略有延迟 | ✅ 可接受 |
| 2 | 无缩略图时加载完整图浪费流量 | 每个SKU最多2张图（库区+箱图），且 Coil 磁盘缓存后不重复下载 | ✅ 可接受 |
| 3 | SSL 证书问题 | Coil 已配置 `trustAllClient`（App.kt），完整图加载与缩略图相同 | ✅ |
| 4 | `areaImageUrl` 为 null 但 `areaThumbUrl` 不为 null | `?:` 不会触发，加载 thumb（虽然文件不存在会404，但 Coil 静默处理） | ✅ 无异常 |
| 5 | 条件检查用 `areaImageUrl`，加载用 `areaThumbUrl` | 逻辑不变，只是新增回退 | ✅ |
