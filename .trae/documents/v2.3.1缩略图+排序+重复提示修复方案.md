# v2.3.1 修复：缩略图显示 + 排序 + 重复添加提示

## 问题 1：库区图/箱图缩略图不能正常显示

### 根因

后端 `_generate_thumbnail()` 始终保存为 `.jpg`：

```python
# images.py L62-63
base, _ = os.path.splitext(file_path)
thumb_path = f"{base}_thumb.jpg"   # 始终 .jpg
```

但前端 `buildThumbUrl()` 保留了原始扩展名：

```kotlin
val dot = fullUrl.lastIndexOf('.')
return "${fullUrl.substring(0, dot)}_thumb${fullUrl.substring(dot)}"
```

当上传 `.png` / `.webp` 格式图片时：
| 上传格式 | DB 记录 | Thumb URL 指向 | 磁盘实际文件 |
|:--------|:--------|:--------------|:------------|
| `.jpg` | `images/xxx.jpg` | `xxx_thumb.jpg` ✅ | `xxx_thumb.jpg` |
| `.png` | `images/xxx.png` | `xxx_thumb.png` ❌ | `xxx_thumb.jpg` |
| `.webp` | `images/xxx.webp` | `xxx_thumb.webp` ❌ | `xxx_thumb.jpg` |

### 修复

**文件**：[PickDetailViewModel.kt](file:///d:/trea项目/快麦取货通/app/src/main/java/com/kuaimai/pda/ui/pickdetail/PickDetailViewModel.kt)

修改 `buildThumbUrl()`，始终替换为 `_thumb.jpg`：

```kotlin
private fun buildThumbUrl(fullUrl: String?): String? {
    if (fullUrl == null) return null
    val dot = fullUrl.lastIndexOf('.')
    return if (dot > 0) "${fullUrl.substring(0, dot)}_thumb.jpg" else "$fullUrl._thumb.jpg"
}
```

## 问题 2：取货单详情列表排序 + 新商品自动滚到顶部

### 现状

排序逻辑正确（[PickDetailScreen.kt L181-185](file:///d:/trea项目/快麦取货通/app/src/main/java/com/kuaimai/pda/ui/pickdetail/PickDetailScreen.kt#L181-L185)）：
- 未完成在上，已完成在下
- 同状态按时间倒序（新添加的在最上方）

### 问题

扫描添加新商品后，`LazyColumn` 保持当前位置，不自动滚到顶部。如果之前滚动到底部，新添加的商品被遮挡。

### 修复

**文件**：[PickDetailScreen.kt](file:///d:/trea项目/快麦取货通/app/src/main/java/com/kuaimai/pda/ui/pickdetail/PickDetailScreen.kt)

在扫码成功事件中增加 `listState.animateScrollToItem(0)`：

```kotlin
// 扫码成功反馈 + 清空输入框并重新聚焦 + 滚到顶部
LaunchedEffect(Unit) {
    viewModel.scanSuccessEvent.collectLatest {
        viewModel.provideFeedback(context, ScanFeedbackType.SUCCESS)
        scanInput = ""
        focusRequester.requestFocus()
        listState.animateScrollToItem(0)  // ← 新增：滚到列表顶部
    }
}
```

## 问题 3：重复添加商品提示不友好

### 现状

重复扫码本地检测后，弹出底部 Snackbar：

```kotlin
snackbarHostState.showSnackbar("重复扫码！该SKU已在当前取货单中")
```

用户希望改为屏幕中间弹窗，自动消失，更友好直观。

### 修复

**文件**：[PickDetailScreen.kt](file:///d:/trea项目/快麦取货通/app/src/main/java/com/kuaimai/pda/ui/pickdetail/PickDetailScreen.kt)

#### 修改 1：新增状态变量

在 `previewPicImageUrl` 之后增加：

```kotlin
var showDuplicateTip by remember { mutableStateOf(false) }
var duplicateTipText by remember { mutableStateOf("") }
```

#### 修改 2：重复扫码触发改为设置居中弹窗

```kotlin
LaunchedEffect(duplicateScan) {
    if (duplicateScan) {
        viewModel.provideFeedback(context, ScanFeedbackType.DUPLICATE)
        showDuplicateTip = true
        duplicateTipText = "该商品已在列表中"
        val duplicateSku = viewModel.lastScannedSku
        if (duplicateSku.isNotEmpty()) {
            val duplicateIndex = items.indexOfFirst { it.skuOuterId == duplicateSku }
            if (duplicateIndex >= 0) {
                listState.animateScrollToItem(duplicateIndex)
            }
        }
        kotlinx.coroutines.delay(1500)
        showDuplicateTip = false
        viewModel.clearDuplicateScan()
    }
}
```

#### 修改 3：添加居中弹窗覆盖层

在图片大图预览弹窗之前添加：

```kotlin
// 重复添加提示 - 居中弹窗，1.5秒后自动消失
if (showDuplicateTip) {
    AlertDialog(
        onDismissRequest = { },
        shape = RoundedCornerShape(16.dp),
        title = null,
        text = {
            Column(
                modifier = Modifier
                    .padding(horizontal = 16.dp, vertical = 8.dp)
                    .fillMaxWidth(),
                horizontalAlignment = Alignment.CenterHorizontally
            ) {
                Text("✓", fontSize = 36.sp, color = SuccessText)
                Spacer(modifier = Modifier.height(12.dp))
                Text(duplicateTipText, fontSize = 16.sp, fontWeight = FontWeight.Medium)
            }
        },
        confirmButton = {}
    )
}
```

`onDismissRequest = { }` 阻止手动关闭，仅靠 `delay(1500)` 自动消失。

**注意**：`Text` 中使用 `fontWeight = android.text.font.FontWeight.Medium`（限定名避免新增 import）。

#### 修改 4：删除原 LaunchedEffect 中的 snackbar 调用

原 LaunchedEffect 块中不再调用 `snackbarHostState.showSnackbar(...)`，以免冲突。

## 回归风险

| # | 风险 | 说明 | 等级 |
|:-:|:-----|:------|:----:|
| 1 | URL 包含额外 `.`（如 `xxx.v2.jpg`） | `lastIndexOf('.')` 找到最后一个 `.`，`_thumb.jpg` 替换正确 | ✅ |
| 2 | URL 有查询参数（如 `?t=123`） | 本项目 URL 由服务端生成，无查询参数 | ✅ |
| 3 | 首次滚动到底部时新商品在顶部 | `animateScrollToItem(0)` 确保滚动到第一个 item | ✅ |
| 4 | 其他 ViewModel 也使用 `scanSuccessEvent` | 仅在 PickDetailScreen 中的 `collectLatest` 块内调用 | ✅ |
| 5 | AlertDialog 手动关闭 | `onDismissRequest = { }` 阻止关闭，仅 `delay(1500)` 消失 | ✅ |
| 6 | `snackbarHostState.showSnackbar` 不再用于重复提示 | 原位置已用 `showDuplicateTip` 替代，无残留调用 | ✅ |
| 7 | Room Flow 异步可能导致 scroll 时新商品未就位 | `animateScrollToItem(0)` 在 Room 更新前后都安全（旧索引0 或 新索引0 都是顶部） | ✅ |
