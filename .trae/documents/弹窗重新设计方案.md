# 弹窗重新设计方案

## 一、概要

重新设计两个弹窗：确认弹窗（ConfirmDialog）和拣货区选择弹窗（NewOrderDialog），提升视觉质感和交互体验。

---

## 二、当前状态分析

### 2.1 ConfirmDialog — 当前 [ProductScreen.kt](file:///d:/trea项目/快麦取货通/app/src/main/java/com/kuaimai/pda/ui/product/ProductScreen.kt#L664-L742)

**当前问题**：
- 标题/消息/高亮内容三块堆叠，视觉层次不够清晰
- 高亮内容使用 Card 组件包裹，但卡片内边距偏小（12dp），高亮信息不够突出
- 确认按钮和取消按钮样式普通，缺乏区分度
- 整体设计偏平，缺乏深度感和精致感

**当前样式**：
```
┌─────────────────────────┐
│     确认切换供应商        │  ← 18sp Bold, TextPrimary
│                          │
│ 是否将供应商切换为        │  ← 14sp, TextSecondary
│  ┌─────────────────────┐ │
│  │     备货①           │ │  ← 20sp Bold, centered
│  └─────────────────────┘ │
│                          │
│         [取消] [确认修改]  │
└─────────────────────────┘
```

### 2.2 NewOrderDialog — 当前 [PickListScreen.kt](file:///d:/trea项目/快麦取货通/app/src/main/java/com/kuaimai/pda/ui/picklist/PickListScreen.kt#L255-L312)

**当前问题**：
- 标题"新建取货单"样式过于简朴（默认 typography）
- "选择拣货区："文字使用 bodyMedium，不够醒目
- 拣货区按钮使用全宽 Button，样式与主操作按钮混淆
- 加载状态用 CircularProgressIndicator 居中，无文案说明
- 空状态文案"暂无拣货区，请先在设置中配置"样式不突出
- 整体缺乏视觉层次和细节打磨

**原型参考**（[prototype/index.html](file:///d:/trea项目/快麦取货通/prototype/index.html#L1280-L1287)）：
```
┌──────────────────────┐
│     选择拣货区         │  ← 18px, 700, centered
│                       │
│  [拣货区A]  [拣货区B]  │  ← grid, 2px border, min-width 100px
│  [拣货区C]  [拣货区D]  │  ← hover: 蓝边框+浅蓝底
│                       │
│       [取消]           │  ← 灰底, 14px
└──────────────────────┘
```

## 三、设计方案

### 3.1 ConfirmDialog 重新设计

| 元素 | 当前 | 新设计 |
|:----|:-----|:-------|
| 标题 | 18sp Bold | 20sp Bold + 顶部间距增大 |
| 高亮内容 | Card 白底 + 12dp 内边距 | Card 浅蓝底 + 16dp 内边距 + 加阴影 |
| 确认按钮 | "确认修改" PrimaryLightBg/PrimaryLightText（浅底深字） | "确认切换" **BrandBlue/SurfaceWhite（品牌蓝底白字）** |
| 取消按钮 | TextButton 灰字 | TextButton 品牌蓝字 |
| 间距 | 紧凑 | 加大上下间距，呼吸感 |

**新样式**：
```
┌──────────────────────────────┐
│     确认切换供应商            │  ← 20sp Bold
│                              │
│  确认将供应商切换为            │  ← 15sp, TextSecondary
│                              │
│  ┌──────────────────────────┐│
│  │       备货①               ││  ← 20sp Bold, 16dp padding
│  │  供应商编码: 0001         ││  ← 新增编码展示, 13sp
│  └──────────────────────────┘│
│                              │
│          [取消]   [确认切换]   │  ← 取消蓝字/确认品牌蓝底白字
└──────────────────────────────┘
```

### 3.2 NewOrderDialog 重新设计（原型风格）

按照原型 design 重新设计（[#L1280-L1329](file:///d:/trea项目/快麦取货通/prototype/index.html#L1280-L1329)）：

| 元素 | 当前 | 新设计（原型风格） |
|:----|:-----|:-----------------|
| 标题 | "新建取货单" 默认样式 | "选择拣货区" 18sp Bold centered |
| 布局 | Column 列表 | **Flex-wrap 网格**（每行2-3个） |
| 按钮样式 | 全宽 Button（PrimaryLightBg） | bordered button：2dp 边框 + 透明背景，hover → 蓝边框+浅蓝底 |
| 取消按钮 | TextButton "取消" | 独立 Cancel button（SurfaceGray 背景，TextSecondary 字色） |
| 加载态 | CircularProgressIndicator 居中 | CircularProgressIndicator + "加载中…" |
| 空状态 | 灰色文字提示 | 浅黄色警告卡片样式 |

**新样式（原型风格）**：
```
┌──────────────────────────────┐
│        选择拣货区              │  ← 18sp Bold, centered
│                              │
│  ┌────────┐  ┌────────┐     │
│  │ 拣货区A │  │ 拣货区B │     │  ← Flex-wrap grid
│  └────────┘  └────────┘     │     border 2dp, rounded 8dp
│  ┌────────┐  ┌────────┐     │     min-width 100dp
│  │ 拣货区C │  │ 拣货区D │     │     hover: BrandBlue border + PrimaryLightBg
│  └────────┘  └────────┘     │
│                              │
│           [取消]              │  ← SurfaceGray bg, TextSecondary
└──────────────────────────────┘
```

## 五、假设与决策

| 决策 | 说明 |
|:----|:------|
| ConfirmDialog 确认按钮改为品牌蓝底白字 | 提高视觉权重，区分于"浅底深字"的次要操作按钮 |
| 供应商弹窗增加编码展示 | 利用 `ConfirmType.Supplier.code` 已有字段，提升信息完整性 |
| NewOrderDialog 跟随原型网格布局 | 使用 Compose `FlowRow` 替代 Column 实现 flex-wrap |
| 加载/空状态增加文案 | 提升用户体验，避免纯加载指示器无信息 |
| 无需拆分新文件 | 两个 dialog 仅各自 Screen 内使用，保持私有 composable 模式 |

## 六、验证步骤

1. `./gradlew lint` 通过
2. 商品详情 → 点击供应商"切换" → 确认弹窗显示新样式（品牌蓝底按钮 + 编码展示）
3. 取货单列表 → 点击"新建" → 弹窗原型风格网格布局
4. 确认弹窗点击"确认" → 供应商切换正常
5. 确认弹窗点击"取消" → 弹窗关闭无操作
6. 拣货区弹窗点击拣货区按钮 → 创建取货单正常
7. 拣货区弹窗点击"取消" → 弹窗关闭
8. 加载中时 → 显示加载中状态
9. 无拣货区时 → 显示空状态提示
