## 三、根因定位

### 根因（唯一）：同步脚本目标路径与 Docker volume mount 不匹配

**完整问题链路：**

```
sync-to-docker-deploy.ps1
  ↓ 第189行: Dst = (Join-Path $DockerDeployRoot "kuaimai.json")
docker-deploy/kuaimai.json     ← 同步脚本放到这里 (根目录)

docker-compose.yml
  ↓ volumes: - ./data:/data     ← Docker 卷挂载映射

容器内 KUAIMAI_CONFIG_PATH:
  /data/kuaimai.json            ← 后端配置路径

实际状况:
  ❌ /data/kuaimai.json 不存在
  ❌ docker-deploy/data/kuaimai.json 不存在
  ❌ backend/data/kuaimai.json 不存在
  ✅ docker-deploy/kuaimai.json 存在 (但容器内 /data/ 看不到)
  ✅ backend/kuaimai.json 存在 (但容器内 /data/ 看不到)
  ↓
load_kuaimai_config() → "快麦凭证文件不存在: /data/kuaimai.json"
  ↓
kuaimai_creds: app_key="" app_secret="" session="" refresh_token=""
  ↓
后端 _call_api() 中 is_valid() 检测失败 → ValueError("快麦凭证未配置")
  ↓
Android GET /api/sku/{skuOuterId} → 502
Android GET /api/kuaimai/credentials → appKey=""（空）
KuaimaiApiService 直连 → Interceptor 发现凭据为空，跳过签名 → 快麦拒接
```

**关键证据文件：**

| 文件 | 行号 | 内容 | 结论 |
|:----|:----:|:-----|:----:|
| [sync-to-docker-deploy.ps1](file:///d:/trea项目/快麦取货通/scripts/sync-to-docker-deploy.ps1#L189) | L189 | `Dst = (Join-Path $DockerDeployRoot "kuaimai.json")` | 同步到根目录 |

| 磁盘实际验证 | — | `backend/data/` 目录不存在 | — |
| 磁盘实际验证 | — | `docker-deploy/data/kuaimai.json` 不存在 | — |
| 磁盘实际验证 | — | `docker-deploy/kuaimai.json` 存在 | 错误位置 |

**这个 bug 一直存在但被掩盖了**：之前系统能运行，是因为 Docker volume 是**持久化**的。旧凭证通过管理后台写入了 `/data/kuaimai.json`（该路径通过 volume mount 映射到 `backend/data/` 或 `docker-deploy/data/`），该文件在容器重启后仍然保留在 volume 中。用户更换凭证时修改了源文件 `backend/kuaimai.json` 但未通过管理后台更新，旧凭证到期后系统暴露出这个路径不匹配问题。

## 四、修复方案

### 4.1 修复同步脚本 — 凭证文件放入 data/ 子目录

**修改文件**：`scripts/sync-to-docker-deploy.ps1` — 仅改 1 行

```powershell
# 修改前（L189）
@{ Name = "kuaimai.json"; Src = ...; Dst = (Join-Path $DockerDeployRoot "kuaimai.json") }

# 修改后
@{ Name = "kuaimai.json"; Src = ...; Dst = (Join-Path $DockerDeployRoot "data/kuaimai.json") }
```

**说明**：PowerShell 的 `Copy-Item -Force` 会自动创建 `data/` 目录（如不存在），无需手动创建目录。

### 4.2 立即恢复凭证加载（通过管理后台）

**前提条件**：拥有 `settings` 权限的管理员账号。管理后台使用 X-API-Key 认证自动拥有全部权限，无需单独配置。

**操作步骤**：
1. 浏览器访问 `http://服务器IP:8900/admin`
2. 登录管理账号（输入 API Key：`zxf199333`）
3. 进入「凭证管理」页面
4. 填入新凭证（以下值来自 `backend/kuaimai.json`）：
   - appKey: `451446590`
   - appSecret: `d3ee57988c754c4eacbdcd25ccca56a1`
   - session: `d43e947e372344fcb460786fc72d69bc`
   - refreshToken: `451e2b42e8104f98b589e3cef65be1bc`
5. 点击保存 → 即时生效（无需重启容器）

**原理**：管理后台的保存操作调用 `save_kuaimai_config()`，它会原子写入 `/data/kuaimai.json`（即 Docker volume 中的持久化文件），然后 `config_watcher` 检测到文件变更后自动重载凭证明到内存。

**定时刷新**：后端已配置每 24 小时自动刷新 session（`main.py` 中 `_refresh_kuaimai_session`），无需手动维护。

### 4.3 PDA 端操作

**前提条件**：PDA 登录用户必须拥有 `settings` 权限，否则 `syncKuaimaiCredentials()` 会收到 403 错误并静默失败。

操作流程：
1. PDA 退出当前登录
2. **确认 PDA 用户的数据库记录包含 `settings` 权限**（可在管理后台 → 用户管理 → 编辑该用户，勾选 `settings`）
3. 重新登录 PDA
4. 自动触发 `syncKuaimaiCredentials()` → 从后端同步新凭证到本地 `EncryptedSharedPreferences`

**权限检查**：`GET /api/kuaimai/credentials` 需要 `settings` 权限（对应 `auth.py` 中 `VALID_PERMISSIONS`）。这是唯一需要确认的权限前置条件。

## 五、回归风险评估

| 风险项 | 评估 | 缓解措施 |
|:-------|:----|:---------|
| 同步脚本 1 行修改引入格式错误 | 极低 | 仅改目标路径字符串，修改后运行一次 `.\scripts\sync-to-docker-deploy.ps1 -Force` 验证 |
| `docker-deploy/data/kuaimai.json` 覆盖 NAS volume 中旧文件 | 不冲突 | volume 挂载覆盖是预期行为，旧文件改为新文件 |
| Android 端需要更新 | **不需要** | 同步机制已在 `UserRepository.syncKuaimaiCredentials()` 中实现 |
| 后端代码需要修改 | **不需要** | 配置加载逻辑、凭证存取逻辑正确 |
| PDA 用户无 `settings` 权限 | 已有 catch 保护，静默失败 | 需要在管理后台手动赋予权限 |
| session 与 appKey 不匹配 | 需通过管理后台手动输入，如果不对可重新获取 | 管理后台支持重新填入 |
| 管理后台无法访问（FRP 不通） | 需排查网络问题 | 可选方案：SSH 到 NAS，直接在 Docker 容器内创建 `/data/kuaimai.json` |
