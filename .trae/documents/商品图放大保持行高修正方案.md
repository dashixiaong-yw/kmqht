# 商品图放大同时保持行高修正方案

## 一、问题

**当前状态**：SKU 图已改为 100×100dp，导致行高从 98dp 增加到 120dp。

用户要求：保持行高 98dp，图片高度降为 90dp，不裁剪，剩下的垂直空间作为间距。

---

## 二、修改内容（PickItemRow.kt — 4 处）

### 修改 1：图片尺寸 100dp 宽 × 90dp 高

**L87** `size(width = 100.dp, height = 100.dp)` → **`size(width = 100.dp, height = 90.dp)`**

```kotlin
// 修改前
.size(width = 100.dp, height = 100.dp)

// 修改后
.size(width = 100.dp, height = 90.dp)
```

- 宽度 100dp 保持 2x 放大效果
- 高度 90dp **自然显示，不裁剪**（删除 `ContentScale.Crop` 保留为 `ContentScale.Crop` 反而刚好 — 90dp 原图刚好填满 90dp 盒子）
  - 实际上 `fillMaxSize()` 配合 90dp 盒子，`ContentScale.Crop` 不会触发裁剪（宽高比一致）

### 修改 2：减小 Row 垂直 padding

**L80** `.padding(horizontal = 12.dp, vertical = 10.dp)` → **`.padding(horizontal = 12.dp, vertical = 4.dp)`**

**计算**：行高 98dp − 图片高 90dp = 8dp → 上下各 4dp ✅

### 修改 3：减小图片与文字间隙

**L104** `Spacer(width = 12.dp)` → **`Spacer(width = 6.dp)`**

### 修改 4：减小文字与右侧图库/按钮间隙

**L128** `Spacer(width = 12.dp)` → **`Spacer(width = 6.dp)`**

---

### 行高核算

| 元素 | 值 |
|:-----|:--:|
| SKU 图高 | 90dp |
| 垂直 padding | 4dp×2 = 8dp |
| **行高** | **90 + 8 = 98dp ✅** |

---
