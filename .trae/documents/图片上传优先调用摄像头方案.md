# 图片上传优先调用摄像头方案

## 一、问题描述

当前库区图和装箱图上传仅支持从图库选择图片（`PickVisualMedia`）。需要在拍照上传时优先调用系统摄像头拍照，当设备没有摄像头时才改用图库选择。

## 二、当前状态分析

### 2.1 现有基础设施（无需修改）

| 项目 | 状态 | 说明 |
|:-----|:-----|:------|
| `CAMERA` 权限 | ✅ 已声明 | `AndroidManifest.xml` L15 |
| Camera 硬件 feature | ✅ 已声明 `required="false"` | `AndroidManifest.xml` L16 |
| `FileProvider` | ✅ 已配置 | `AndroidManifest.xml` L42-50，`file_paths.xml` |
| `cache-path` 可用 | ✅ 已暴露 | `file_paths.xml` L4: `<cache-path name="sync_logs" path="/" />` |
| `uploadImage(file, type)` | ✅ 已存在 | `ProductViewModel.kt` L395，接收 `File` 参数 |

### 2.2 当前图片上传流程

```
用户点击"上传库区图/装箱图"
  └─ pendingImageType = IMAGE_TYPE_AREA/BOX
  └─ pickImageLauncher.launch(PickVisualMediaRequest(ImageOnly))  // 图库选择
  └─ onResult: uri → uriToFile(uri) → viewModel.uploadImage(file, type)
```

```kotlin
val cameraLauncher = rememberLauncherForActivityResult(
    TakePicture()

    if (success) {
```

#### 变更C — 修改上传按钮点击逻辑（L225-232 区域）

**修改前：**

**修改后：**

`onUploadArea` 和 `onUploadBox` 的 onClick 逻辑：检查摄像头 → 优先拍照 → 失败或无摄像头时回退到图库。

```kotlin
onUploadArea = {
    val pm = context.packageManager
    if (pm.hasSystemFeature(PackageManager.FEATURE_CAMERA_ANY)) {
        try {
            val photoFile = createTempImageFile(context)
            pendingImageType = AppConstants.IMAGE_TYPE_AREA
            pendingCameraFile = photoFile
            cameraLauncher.launch(
                FileProvider.getUriForFile(context, "${context.packageName}.fileprovider", photoFile)
            )
        } catch (_: ActivityNotFoundException) {
            pendingImageType = AppConstants.IMAGE_TYPE_AREA
            pickImageLauncher.launch(PickVisualMediaRequest(ActivityResultContracts.PickVisualMedia.ImageOnly))
        } catch (_: Exception) {
            pendingImageType = AppConstants.IMAGE_TYPE_AREA
            pickImageLauncher.launch(PickVisualMediaRequest(ActivityResultContracts.PickVisualMedia.ImageOnly))
        }
    } else {
        pendingImageType = AppConstants.IMAGE_TYPE_AREA
        pickImageLauncher.launch(PickVisualMediaRequest(ActivityResultContracts.PickVisualMedia.ImageOnly))
    }
},
onUploadBox = {
    val pm = context.packageManager
```

#### 变更D — 新增 createTempImageFile 工具函数（文件底部，uriToFile 附近）

```kotlin
/**
 * 创建临时图片文件（用于相机拍照保存）
 */
private fun createTempImageFile(context: Context): File {
    val timeStamp = System.currentTimeMillis()
    return File.createTempFile("capture_${timeStamp}_", ".jpg", context.cacheDir)
}
```

#### 变更E — pickImageLauncher 保持回退路径不变

`pickImageLauncher`（L134-145）和 `uriToFile`（L800-816）保持原样，作为无摄像头时的降级路径。

## 四、流程示意

```
用户点击"上传库区图/装箱图"
  ├─ PackageManager.hasSystemFeature(FEATURE_CAMERA_ANY)?
  │  ├─ YES → createTempImageFile → FileProvider.getUriForFile
  │  │        → TakePicture.launch(photoUri)
  │  │        └─ 拍照成功 → viewModel.uploadImage(file, type)
  │  │        └─ 拍照失败 → 静默结束（pendingImageFile/pendingImageType 清空）
  │  └─ NO → pickImageLauncher(PickVisualMedia)  ← 回退到图库
```

## 五、回归风险评估

| 风险点 | 评估 |
|:-------|:-----|
| 无摄像头设备的图库选择 | 仅当 `FEATURE_CAMERA_ANY` 为 false 时走图库，完全保留现有逻辑 |
| `ActivityNotFoundException` | 已在 onClick try-catch 中捕获，捕获后回退到图库 |
| 拍照后取消（返回 false） | 仅清理 pending 状态，无副作用 |
| `FileProvider.getUriForFile` 失败 | 已在 try-catch 中，回退到图库 |
| 修改范围小 | 仅改 1 个文件，不影响其他功能模块 |
