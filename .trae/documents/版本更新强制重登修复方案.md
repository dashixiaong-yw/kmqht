# 强制更新/安装后弹窗修复方案（v2.3）

## 一、根因确认

### 真实的触发链

```
APK更新安装后 → 旧 token 仍存在于 EncryptedSharedPreferences 中
  → restoreFromCache() 读取旧 token
    → 但旧 token 可能已经过期（7天周期）或加密数据未能正确解密
    → _cachedToken = 旧/空的 token
  → 用户操作: 点击取货列表 / 扫码获取商品信息
    → API 调用 → getToken() → 返回空值或过期值
    → 后端验证 X-User-Token → 不存在/过期 → HTTP 401

OkHttp TokenAuthenticator 接收到 401
  → 尝试 systemApiService.refreshSession(userToken)
    → POST /api/kuaimai/refresh-session
    → 需要有效的用户 token (X-User-Token 头)
    → token 为空/过期 → 后端返回 HTTP 401 或 403
    → Retrofit 抛 HttpException → catch → null (刷新失败)
  → notifySessionExpired()
    → AppNavigation 弹窗: "快麦API会话已过期，请在Web管理后台重新授权"

重新登录后:
  → login() → 获取新 token（7天有效）
  → syncKuaimaiCredentials(token) → 获取新快麦凭证
  → _cachedToken = newToken
  → 所有 API 正常 → 无弹窗
```

### 为什么 v2.1 `_cachedToken` 不够

`_cachedToken` 解决了 **登录后** 读取 SP 的时序问题，但没有解决 **更新安装后旧 token 本身已过期** 的问题：

| Token 状态 | v2.1 之前 | v2.1 (_cachedToken) | v2.3 (本方案) |
|:-----------|:----------:|:--------------------:|:--------------:|
| 登录后读取（时序问题） | ❌ 空值 | ✅ 内存缓存 | ✅ 内存缓存 |
| 更新后旧token过期 | ❌ 弹窗 | ❌ 仍然弹窗 | ✅ 强制重登 |

---

## 二、方案

### 核心思路

在 App 启动时检查版本号是否变化，若变化则清除用户数据（包括快麦凭证），强制跳转登录页，让用户重新登录获取新 token + 快麦凭证。

### 修改点

#### 文件 1：`PrefsKeys.kt` — 新增 storage key

```kotlin
/** 上次运行的应用版本号 */
const val KEY_LAST_VERSION_CODE = "last_version_code"
```

#### 文件 2：`UserRepositoryImpl` — 新增版本变更检测

**新增字段**（与 `_cachedToken` 相邻）：

```kotlin
/** 应用是否刚升级（版本号变化） */
private var isVersionChanged: Boolean = false
```

**修改 `restoreFromCache()`**：

```kotlin
private fun restoreFromCache() {
    // 检测版本号是否变化
    val lastVersion = prefs.getInt(PrefsKeys.KEY_LAST_VERSION_CODE, 0)
    val currentVersion = BuildConfig.VERSION_CODE
    if (lastVersion > 0 && lastVersion != currentVersion) {
        // 版本已更新 → 清除所有用户数据，强制重新登录
        Log.i(TAG, "版本更新: $lastVersion → $currentVersion，清除用户数据")
        clearLocalUser()
        // 额外清除快麦凭证（clearLocalUser 不清除这些）
        prefs.edit()
            .remove(PrefsKeys.KEY_APP_KEY)
            .remove(PrefsKeys.KEY_APP_SECRET)
            .remove(PrefsKeys.KEY_SESSION)
            .apply()
        _cachedToken = ""  // 确保内存缓存也被清除
        isVersionChanged = true
        return  // 跳过后续恢复
    }
    
    // 保存当前版本号（无论是否升级都更新）
    prefs.edit().putInt(PrefsKeys.KEY_LAST_VERSION_CODE, currentVersion).apply()
    
    // 原有恢复逻辑...
    val token = prefs.getString(PrefsKeys.KEY_USER_TOKEN, "") ?: ""
    ...
}
```

**新增判断方法**：

```kotlin
/** 是否因版本更新需要强制登录 */
fun isVersionChanged(): Boolean = isVersionChanged
```

**修改 `login()`**（登录成功后重置标志）：

```kotlin
// 在 isVersionChanged 使用后重置
isVersionChanged = false  // 可放在 Result.success 之前
```

**修改 `clearLocalUser()`**（补充清除快麦凭证，因它们存续会导致旧 session 残留）：

```kotlin
private fun clearLocalUser() {
    _cachedToken = ""
    prefs.edit()
        .remove(PrefsKeys.KEY_USER_TOKEN)
        .remove(PrefsKeys.KEY_USER_ID)
        .remove(PrefsKeys.KEY_USER_NAME)
        .remove(PrefsKeys.KEY_USER_PERMISSIONS)
        .remove(PrefsKeys.KEY_SESSION_EXPIRE)
        .remove(PrefsKeys.KEY_APP_KEY)      // ← 新增
        .remove(PrefsKeys.KEY_APP_SECRET)    // ← 新增
        .remove(PrefsKeys.KEY_SESSION)       // ← 新增
        .apply()
    _currentUser.value = null
    Log.i(TAG, "本地用户数据已清除")
}
```

#### 修改 3：`UserRepository.kt` 接口 — 新增方法（可选）

```kotlin
/** 判断是否刚完成版本更新（需重新登录） */
fun isVersionChanged(): Boolean
```

#### 修改 4：`AppNavigation.kt` — 版本变更时强制登录

```kotlin
// 启动时验证token有效性（在 L80-87 处）
LaunchedEffect(Unit) {
    // 版本更新 → 直接跳转登录
    if (userRepository.isVersionChanged()) {
        startDestination = Routes.LOGIN
    } else if (userRepository.isLoggedIn() && userRepository.isTokenLocallyValid()) {
        startDestination = Routes.HOME
    } else {
        startDestination = Routes.LOGIN
    }
    isCheckingAuth = false
}
```

### 本次修补完整性验证

#### 数据流验证

```
安装新版本 APP（2.3）:
  → restoreFromCache()
    → lastVersion (2.1) ≠ currentVersion (2.3)
    → prefs.edit().remove(KEY_USER_TOKEN...KEY_SESSION...).apply()
    → clearLocalUser()
    → _cachedToken = ""
    → isVersionChanged = true
    → return (跳过恢复)

  → AppNavigation LaunchedEffect
    → isVersionChanged() == true
    → startDestination = Routes.LOGIN

用户登录:
  → login() 成功
  → _cachedToken = newToken
  → syncKuaimaiCredentials(newToken) → 获取快麦凭证
  → isVersionChanged = false

所有 API 调用:
  → getToken() → _cachedToken = newToken（7天有效）
  → 后端 200 OK
  → 无 401 → TokenAuthenticator 不触发 → 无弹窗 ✅
```

#### 回归风险

| # | 风险 | 预案 |
|:-:|:-----|:------|
| 1 | `lastVersion == 0`（首次安装） | `> 0` 条件保护，首次安装不走清除逻辑 |
| 2 | 升级后老用户需重新登录 | 预期行为，只一次，后面正常 |
| 3 | `prefs.edit()...remove().apply()` 时序 | `_cachedToken = ""` 内存同步确保立即生效 |
| 4 | `isVersionChanged` 竞态 | 单线程 `LaunchedEffect` 读取后自动重置 |
| 5 | 旧 token 未过期但被迫重登 | 只有小版本推更时才触发，短期小版本推更频率低 |

## 四、验证步骤

1. 安装旧版本 → 登录 → 使用正常
2. 安装 v2.3 APK（升级安装）
3. APP 启动 → 弹出登录页（自动，无需用户操作）
4. 使用当前账号重新登录
5. 点击取货列表 → 正常加载，无弹窗
6. 扫码获取商品信息 → 正常
7. 退出 → 重新进入 → 不再要求登录（正常保持登录态）
