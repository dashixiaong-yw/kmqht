# 启动时更新弹窗修复方案

## 一、问题分析

### 症状

管理后台点击"分发"后，PDA App 不会弹出任何更新提示弹窗。

### 根因

`MainActivity.kt` L65-L80 的启动时自动检查更新代码存在设计缺陷：

```kotlin
// 当前代码：发现新版本后静默下载+安装，无用户提示
lifecycleScope.launch {
    when (val result = appUpdateManager.checkForUpdate()) {
        is CheckResult.HasUpdate -> {
            appUpdateManager.downloadApk(result.info)  // 静默下载
            launch {
                appUpdateManager.downloadState.collect { state ->
                    if (state is DownloadState.Completed) {
                        appUpdateManager.installApk(state.file)  // 静默安装
                    }
                }
            }
        }
        else -> {}
    }
}
```

**对比 SettingsScreen 的正确流程**（[SettingsScreen.kt L145-L170](file:///d:/trea项目/快麦取货通/app/src/main/java/com/kuaimai/pda/ui/settings/SettingsScreen.kt#L145-L170)）：

```
手动检查 → 弹出 AlertDialog "发现新版本" → 用户点击"立即更新" → 下载 → 安装
```

而启动时自动检查的流程是：

```
启动 → 发现有新版本 → 静默下载 → 下载完成 → 静默安装（Android 系统安装器或许会弹窗）
```

**没有 App 级别的弹窗提示用户"有新版本可用"**。

## 二、修改内容

### 唯一改动：`MainActivity.kt`

**文件**：`app/src/main/java/com/kuaimai/pda/MainActivity.kt`

**改动 A**：新增以下 import（当前文件中没有任何 Compose 导入）

```kotlin
import androidx.compose.foundation.layout.Column
```

**改动 B**：在 `setContent` 内部的 `AppNavigation` 旁添加 `LaunchedEffect` 检查和 `AlertDialog` 弹窗

```kotlin
setContent {
    KuaimaiTheme {
        // 启动时自动检查更新弹窗状态
        var showUpdateDialog by remember { mutableStateOf(false) }
        var updateInfo by remember { mutableStateOf<AppVersionResponse?>(null) }
        var isDownloading by remember { mutableStateOf(false) }

        // 启动时自动检查更新
        LaunchedEffect(Unit) {
            when (val result = appUpdateManager.checkForUpdate()) {
                is CheckResult.HasUpdate -> {
                    updateInfo = result.info
                    showUpdateDialog = true
                }
                else -> {}
            }
        }

        // 更新弹窗
        if (showUpdateDialog && updateInfo != null) {
            val info = updateInfo!!
            AlertDialog(
                onDismissRequest = {
                    if (!info.forceUpdate && !isDownloading) {
                        showUpdateDialog = false
                        updateInfo = null
                    }
                },
                shape = RoundedCornerShape(16.dp),
                title = { Text("发现新版本") },
                text = {
                    Column {
                        Text("最新版本: v${info.latestVersion}")
                        if (info.updateNotes.isNotBlank()) {
                            Spacer(modifier = Modifier.height(8.dp))
                            Text(info.updateNotes)
                        }
                        if (isDownloading) {
                            Spacer(modifier = Modifier.height(8.dp))
                            LinearProgressIndicator(modifier = Modifier.fillMaxWidth())
                            Spacer(modifier = Modifier.height(4.dp))
                            Text("正在下载更新...", style = MaterialTheme.typography.bodySmall)
                        }
                    }
                },
                confirmButton = {
                    TextButton(
                        onClick = {
                            if (!isDownloading) {
                                isDownloading = true
                                appUpdateManager.downloadApk(info)
                                // 监听下载完成（lifecycleScope 绑 Activity 生命周期，弹窗关闭后继续）
                                lifecycleScope.launch {
                                    appUpdateManager.downloadState.collect { state ->
                                        if (state is DownloadState.Completed) {
                                            appUpdateManager.installApk(state.file)
                                            showUpdateDialog = false
                                            updateInfo = null
                                        } else if (state is DownloadState.Failed) {
                                            isDownloading = false
                                        }
                                    }
                                }
                            }
                        },
                        enabled = !isDownloading
                    ) {
                        Text(if (isDownloading) "下载中..." else "立即更新")
                    }
                },
                dismissButton = {
                    if (!info.forceUpdate) {
                        TextButton(
                            onClick = {
                                showUpdateDialog = false
                                updateInfo = null
                            }
                        ) {
                            Text("稍后再说")
                        }
                    }
                }
            )
        }

        AppNavigation(...)  // 不变
    }
}
```

### 完整修改前后对比

**修改前** `MainActivity.kt`：

```kotlin
setContent {
    KuaimaiTheme {
        AppNavigation(...)
    }
}
// 启动时自动检查应用更新
lifecycleScope.launch {
    when (val result = appUpdateManager.checkForUpdate()) {
        is CheckResult.HasUpdate -> {
            appUpdateManager.downloadApk(result.info)
            launch { /* 监听下载完成 */ }
        }
        else -> {}
    }
}
```

**修改后** `MainActivity.kt`：

```kotlin
setContent {
    KuaimaiTheme {
        // 弹窗状态
        var showUpdateDialog by remember { mutableStateOf(false) }
        var updateInfo by remember { mutableStateOf<AppVersionResponse?>(null) }
        var isDownloading by remember { mutableStateOf(false) }

        // 检查更新
        LaunchedEffect(Unit) {
            when (val result = appUpdateManager.checkForUpdate()) {
                is CheckResult.HasUpdate -> {
                    updateInfo = result.info
                    showUpdateDialog = true
                }
                else -> {}
            }
        }

        // 弹窗
        if (showUpdateDialog && updateInfo != null) {
            // ... AlertDialog ...
        }

        AppNavigation(...)  // 不变
    }
}
// 不再有 lifecycleScope.launch 外部调用
```

## 三、回归风险分析

| 风险点 | 分析 | 等级 |
|:-------|:-----|:----:|
| SettingsScreen 弹窗不受影响 | 独立 ViewModel + AlertDialog，与 MainActivity 无关 | 低 |
| AppUpdateManager 不受影响 | 仅调用方从 lifecycleScope 改为 Compose scope，Manager 无修改 | 低 |
| 网络异常 | LaunchedEffect 中的 try-catch 已由 AppUpdateManager.checkForUpdate() 提供 | 低 |
| 弹窗被覆盖或丢失 | 纯 Compose AlertDialog，用户"稍后再说"后下次启动再检查 | 低 |
| 重复检查 | LaunchedEffect(Unit) 仅在 Composable 第一次进入时执行，Activity 重建（如屏幕旋转）会重新检查，但这是正常行为 | 低 |
| 强更弹窗不可关闭 | forceUpdate=true 时 onDismissRequest 不执行、dismissButton 不显示，与 SettingsScreen 行为一致 | 预期行为 |

## 四、验证步骤

1. 修改代码后运行 `./gradlew lint` 验证无警告
2. `./gradlew assembleRelease` 构建 APK
3. 管理后台上传新版本 APK 并分发
4. PDA 完全关闭 App 后重启（不要在后台恢复）
5. 观察是否弹出"发现新版本"弹窗
6. 测试"立即更新" → 下载 → Android 安装器弹出
7. 测试"稍后再说" → 弹窗关闭，App 正常使用
8. 测试"强制更新"场景（forceUpdate=true）：弹窗不可关闭
9. Settings 页面的"检查更新"按钮仍正常工作
