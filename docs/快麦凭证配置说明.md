# 快麦开放平台凭证配置说明

## 一、凭证在哪里

快麦凭证仅存在于以下 **2个文件**，其他地方无需修改：

| 文件 | 说明 | 是否提交Git |
|:----|:-----|:----------:|
| `backend/kuaimai.json` | 后端运行时会加载的真实凭证文件 | ❌ 已加入 `.gitignore` |
| `docker-deploy/kuaimai.json` | 部署产物，由同步脚本从 `backend/kuaimai.json` 复制 | ❌ `docker-deploy/` 整体忽略 |

## 二、凭证同步到PDA的流程

```
后端 kuaimai.json (服务器)
    │
    ├── 后端程序启动时 ──→ 加载到内存 (config.kuaimai_creds)
    │                          │
    │                          ├── 调用快麦API时做签名 (kuaimai_api._sign)
    │                          │
    │                          └── PDA登录成功后 GET /api/kuaimai/credentials
    │                                 │
    │                                 └── 写入PDA本地 encryptedPrefs
    │                                        │
    │                                        └── KuaimaiInterceptor 使用本地凭证
    │
    ├── Docker部署: docker-deploy/kuaimai.json (同步脚本复制)
    │
    └── 管理后台: /admin → 凭证管理 → 可在线修改
```

## 三、正式上线时如何替换

### 方法一：直接修改文件（推荐）

1. 打开 `backend/kuaimai.json`（`docker-deploy/kuaimai.json` 在运行同步脚本后自动覆盖）
2. 替换以下4个值：

```json
{
  "app_key": "正式环境的appKey",
  "app_secret": "正式环境的appSecret",
  "session": "正式环境的accessToken/session",
  "refresh_token": "正式环境的refreshToken",
  "updated_at": "2026-xx-xx xx:xx:00"
}
```

3. 重新构建Docker并部署：`docker-compose up -d --build`
4. 后端会自动加载新凭证，无需其他操作

### 方法二：管理后台在线修改（部署后）

1. 浏览器访问 `http://服务器IP:8900/admin`
2. 使用管理账号登录
3. 进入 **凭证管理** 页面
4. 填入新的 appKey / appSecret / session / refreshToken
5. 点击保存，即时生效（无需重启）

### 方法三：替换文件后热加载

1. 服务器上替换 `/data/kuaimai.json` 文件内容
2. 后端自动检测文件变更（watchfiles 热重载功能）
3. 无需重启，约1-2秒内生效

## 五、凭证更新后PDA如何处理

PDA端凭证是在 **用户登录时自动同步** 的：

1. PDA用户登录后端
2. 后端返回 `token`
3. PDA自动调用 `GET /api/kuaimai/credentials` 获取最新凭证
4. 写入本地 `EncryptedSharedPreferences`

> **如果只在后端更新了凭证，PDA不会自动收到新凭证**。PDA用户需要：
> - 退出登录 → 重新登录 → 自动同步新凭证
> 或者
> - 重启App → 如果本地已有有效token，不会重新同步 → 需退出登录再登入

## 七、常见问题

**Q: 修改了 kuaimai.json 需要重启服务吗？**
A: 不需要。后端有文件监控热重载。如果修改后未生效，可等待1-2秒或检查日志中的 `"快麦凭证加载成功"` 消息。

**Q: PDA端提示"快麦凭证未配置"？**
A: 需要用户在PDA上退出登录后重新登录，触发凭证同步。

**Q: refreshToken 过期了怎么办？**
A: 进入管理后台 `/admin` → 凭证管理 → 填入新的 appKey/appSecret/session/refreshToken → 保存。然后在所有PDA上重新登录。

**Q: 凭证信息会不会泄漏到Git？**
A: 不会。`kuaimai.json` 已加入 `.gitignore`，`backend/.dockerignore` 也排除了该文件。只有 `kuaimai.example.json`（占位模板）会提交。
